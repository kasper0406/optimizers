# Program #16 — process-matched tail readout: Phase A pre-registration

2026-07-22, committed before any run. Motivation: the late phase mostly
pays for settling oscillation (measured rho1 = -0.3..-0.5, corpus-wide);
averaging anti-correlated iterates settles faster than 1/n; readout
averaging is OPEN-LOOP, so program #9's endogeneity trap structurally
cannot apply (the trajectory is generated by untouched Muon). Unclaimed
slice per the #9 sweep: readout kernels matched to measured iterate
autocovariance (published averaging = EMA/uniform; Wiener-flavored work
= gradient-side). Known neighbors to beat/match: plain tail-EMA
(speedrun practice), WSM / checkpoint-merging-replaces-decay
(2403.19390), schedule-free.

## Phase A (instrument + offline; the cheap kill-test)

Runs: B = 1000, lr 0.24, seeds 1506-1510 (n = 5), full 8-epoch schedule,
saving per-step weight snapshots from step 80 onward (recipe.save_iterates;
~6 MB x 120 steps x 5 runs ~ 3.6 GB scratch, deleted after analysis).
Additionally the same 5 seeds at a TRUNCATED schedule (5 epochs, its own
full anneal) as the "shorter tuned schedule" reference arm. 10 runs.

Offline: for every truncation point T in {100,...,195} and kernels
{last-iterate, uniform-k, EMA(beta), rho-matched min-variance k-tap
(program #9 synthesize_taps machinery, tau swept)}: evaluate tta accuracy
of the readout average of the FULL-schedule run's iterates up to T
(BN stats recomputed). Note the full-schedule run's lr at step T is the
un-annealed schedule value - this measures "stop early + average"
against "anneal to the end".

## Pre-registered measurements

- M1 (headroom): best readout accuracy at T = 160 (80% budget) vs the
  full-run final accuracy. PASS if within 0.15pp (averaging replaces the
  last 20% of the anneal); strong pass if it EXCEEDS full-run accuracy.
- M2 (kernel ordering): rho-matched kernel vs uniform vs EMA at matched
  window mean-lag: does process-matching beat the naive kernels by
  >= 2x the seed-sd of the readout accuracy? (The #9 decomposition
  predicts most of the gain is window-family, not matching - stated.)
- M3 (against the tuned short schedule): best readout at T = 125
  (5 epochs of the 8-epoch schedule, un-annealed) vs the 5-epoch
  fully-annealed reference arm. PASS if readout >= reference (averaging
  a truncated hot run beats re-tuning the schedule to the short budget).
- M4 (mechanism): readout-averaged weights' distance to the full-run
  endpoint vs the raw iterate's distance (does averaging move along the
  anneal direction, i.e., down the walls?).
